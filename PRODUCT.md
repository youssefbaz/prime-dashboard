# Product

## Register

product

## Users

Single user (the builder) using this daily as a personal command center. Context: alone at a desk or on a phone, in focused work mode. The user is actively executing a multi-month plan — DS + AWS study roadmap, parallel job search, nutrition tracking, habit building. Every session is purposeful. There is no casual browsing.

## Product Purpose

Prime Dashboard is a personal operating system for executing a high-stakes self-improvement plan. It consolidates what would otherwise be scattered across Notion, spreadsheets, timers, flashcard apps, and job trackers into one dark-mode tool the user opens every morning. Success means the user sits down, sees exactly where they are, and gets to work — no friction, no confusion, no wasted motion.

## Brand Personality

Focused. Disciplined. Relentless.

Voice: direct, terse, no fluff. Never cheerful for the sake of it. Feedback is honest ("You've missed 3 days" is better than "Keep trying!"). Motivation is earned through visible progress, not hollow copy.

The tagline says it best: "Prime is built in silence. Discipline = Freedom."

## References

Linear / Raycast: product-grade craft, precise spacing, restrained palette, fast, confident. Every detail considered — not because it looks nice, but because a tool you use every day needs to feel like a precision instrument. Borrow their density, their restraint, their faith that the user doesn't need hand-holding.

## Anti-references

Corporate BI tool (Tableau, Power BI, generic enterprise dashboards): clinical, gray-heavy, zero personality. Data displayed without a point of view. The kind of interface that could belong to any company or any person. Prime has a specific user with a specific mission — the interface should reflect that.

## Design Principles

1. **The tool disappears into the task.** When Prime works, you're not thinking about the interface — you're logging your weight, reviewing flashcards, checking off tasks. Friction is a bug.

2. **Data with intention.** Every number shown is there because it changes behavior. No decorative metrics, no vanity stats. If a number doesn't make the user do something differently, it earns no screen space.

3. **Earned progress.** XP, streaks, level-ups — these exist because the user earns them. Gamification serves discipline, not dopamine. Celebration should feel proportionate to what was accomplished.

4. **Relentless consistency.** Same rhythm, same spacing, same component language across all 14 pages. A tool you use daily must behave predictably. Inconsistency is cognitive tax.

5. **Speed is respect.** This is opened every morning. It must load fast, navigate instantly, and never stall. Performance is a feature.

## Accessibility & Inclusion

No formal WCAG target for this iteration (personal single-user tool). Maintain readable contrast and usable interactions as a baseline.
